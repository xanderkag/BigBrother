# Корпус документов для прогона

**В git ничего из этой папки не коммитим** — корпус NDA, лежит только на сервере.

`.gitignore` рядом исключает все файлы кроме самого README. Если случайно сюда что-то скопировали — `git status` покажет «nothing to commit», но руками лучше держать корпус **вне репо** (`/opt/parsdocs-corpus/`).

## Минимальный состав

Для load-сценария рекомендуется ≥ 100 файлов с распределением, похожим на прод:

| Тип | Доля | Что считаем |
| --- | --- | --- |
| УПД | ~25 % | таблица позиций, ИНН/КПП валидируется |
| Счёт / счёт-фактура | ~25 % | реквизиты + позиции |
| ТТН / CMR | ~15 % | многостраничные, госномера |
| Акт / спецификация | ~10 % | таблицы |
| Договор / допсоглашение | ~10 % | длинный текст, без таблиц |
| Прочее (платёжка, чек, B/L, ГТД, packing list) | ~15 % | edge-cases |

Поддерживаемые форматы: `pdf`, `jpg`/`jpeg`, `png`, `bmp`, `tif`/`tiff`, `webp`.

## Фиксация манифеста

Перед прогоном:

```bash
source ../lib.sh && manifest_corpus > ../runs/<run_id>/manifest.txt
```

Формат строки: `<sha256>\t<size_bytes>\t<abs_path>` — это даёт точное соответствие «эта версия корпуса = эти результаты».

## Подвыборки

Если хочется прогнать на подмножестве — положите список относительных (от `CORPUS_DIR`) путей в текстовый файл и укажите его в `.env`:

```
CORPUS_LIST=/opt/parsdocs-corpus/lists/load-100.txt
```

`lib.sh::list_corpus` подхватит автоматически.
