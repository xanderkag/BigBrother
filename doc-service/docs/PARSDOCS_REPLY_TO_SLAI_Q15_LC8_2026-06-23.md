## 1. Q15 — контейнер в ТТН / CMR / Акт ✅ ГОТОВО (в проде)

Контейнер теперь извлекается из **ТТН, CMR и Акта** (раньше — только из коносамента B/L).
Читать так же, как для B/L — из канонического `extracted._match_signals.containers`:

```json
"_match_signals": {
  "containers": ["TCLU7654321"],
  "order_refs": ["T-2025-006"],
  ...
}
```

- **Тип:** массив строк ISO 6346 (4 буквы + 7 цифр), нормализованы (upper, без пробелов,
  провалидированы по маске). **Present-only** — ключ есть только если контейнер найден
  (контракт тот же, что был у B/L).
- **Покрытие:** B/L (как было) + ТТН + CMR + Акт. Проверено вживую на всех трёх типах.
- **Надёжность:** извлекается моделью, а где модель промахивается (наблюдали на CMR) —
  добивается детерминированным regex'ом по ISO 6346 из текста документа. Для вас это
  прозрачно: ключ один — `_match_signals.containers`.
- `order_refs` (вы просили «желательно») тоже идёт в `_match_signals.order_refs`
  (массив строк), в т.ч. для Акта.

→ Можно цеплять ТТН/CMR/Акт по грузовой единице через `_match_signals.containers`.

## 2. LC-8 — строки счёта перевозчика (per-line транспортные атрибуты)

Подтверждаю: parsdocs шлёт per-line транспортные атрибуты в `items[]`.

> ⚠️ **Критично — ключ контейнера `it.container_no`, а НЕ `it.container`.**
> В ветке `9169d4c` вы закодили `it.container` — это no-op, поле всегда пустое.
> Поправьте на `it.container_no`.

Полный список per-line ключей в `items[i]` (все present-only):

| Атрибут | Ключ в `items[]` |
|---|---|
| Контейнер (ISO 6346) | **`container_no`** ⚠️ (не `container`) |
| Госномер ТС | `vehicle_plate` |
| № заказа / заявки на перевозку | `order_ref` |
| Маршрут — пункт отправления | `route_from` |
| Маршрут — пункт назначения | `route_to` |
| Дата рейса | `trip_date` |
| № коносамента | `bl_no` |
| № CMR | `cmr_no` |
| № ТТН | `ttn_no` |
| № декларации | `declaration_no` |
| Водитель | `driver_name` |
| Категория (товар / услуга) | `category` |

Нормализация: `vehicle_plate` → канонический РФ-номер (upper, без пробелов),
`container_no` → ISO 6346. На товарных инвойсах эти поля пустые (no-op) — как вы и ожидали.

→ Поправьте `it.container` → `it.container_no`, домапьте остальные по таблице, и строки
счёта перевозчика зацепятся к плечам. Нужен пример webhook payload фрахт-счёта с
заполненными per-line полями — пришлю.
