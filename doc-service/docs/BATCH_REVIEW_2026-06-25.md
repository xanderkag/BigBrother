## Открытые пункты (ТЗ)

### P1 — влияет на качество прямо сейчас
1. **Договоры не распознаются.** Тип `contract` в каталоге есть (17 ключей
   классификации), но реальный договор ушёл в «не определён» → 0 извлечённых
   полей. Нужно: (а) расширить `classification_keywords` у `contract` под реальные
   формулировки заголовков (нужны 1–2 обезличенных образца), (б) убедиться, что у
   `contract` задана `llm_schema` (иначе даже после верной классификации полей не
   будет). Договоры — частый тип в заказах, приоритет высокий.
2. **B/L: контейнеры не доходят до match-signals.** `bill_of_lading` извлекает
   `containers[]` (number / type / seal / вес), НО `number` не проходит маску
   ISO-6346 (`^[A-Z]{4}\d{7}$`) → `collectContainers` их отсекает → у SLAI
   `_match_signals.containers` пуст (хотя `bl_number` доходит). Нужно понять на
   образцах: номера в нестандартном формате (тогда нормализовать/смягчить маску)
   или это неконтейнерные B/L. Критично для привязки по грузовой единице.

### P2 — мониторить на большом объёме
3. **Низкий confidence** у `commercial_invoice` (0.71), `packing_list` (0.74),
   `bill_of_lading` (0.83). На 12 доках не криминал, но на большом батче следить:
   при мисклассах/недоизвлечении — тюнить ключи/схемы по образцам.

### P3 — операционное удобство
4. **Нет «переклассифицировать заново».** `reprocess` держит текущий тип
   (`hint = document_type`), поэтому мискласс через «Перепрогнать» не чинится —
   нужно сбрасывать тип (UPDATE document_type=NULL + reprocess) или перезаливать.
   Удобно бы кнопку «Определить тип заново» на детальной.
