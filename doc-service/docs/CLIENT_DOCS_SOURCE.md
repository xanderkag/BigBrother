# parsdocs — Поддерживаемые документы и возможности
### Материал для клиентской документации и маркетинга (актуально на 2026-06)

> Это «источник правды» по тому, **что сервис распознаёт сегодня**, в каком
> статусе зрелости каждый тип и насколько хорошо вытягиваются табличные данные
> (позиции/массивы). Из этого документа собираются: страница «Документы» на
> сайте, раздел в клиентской документации, онбординг новых клиентов.
>
> Тон — честный: production-типы заявляем как готовые, beta — как «работает,
> дорабатываем», experimental — как «в разработке / по запросу». Не завышаем.

---

## 1. Что делает сервис (1 абзац для лендинга)

parsdocs принимает документ (скан, PDF, фото, Excel/Word), сам определяет его
тип, извлекает поля в структурированный вид (JSON) и возвращает результат с
оценкой достоверности по каждому полю. Бухгалтерия и логистика перестают
вбивать данные вручную — счета, накладные, акты, таможенные и ВЭД-документы
превращаются в готовые записи для учётной системы. Поддерживается разбор
**таблиц позиций** (товары/услуги/грузы), многодокументных файлов и проверка
данных на ошибки (контрольные суммы ИНН, согласованность НДС, форматы).

## 2. Легенда статусов (для клиента)

| Значок | Статус | Что значит для клиента |
|---|---|---|
| 🟢 **Production** | Готов | Обкатан, типизирован, есть проверки качества. Заявляем точность. |
| 🔵 **Beta** | Расширенная поддержка | Работает в проде, активно дорабатываем по реальным данным. Подходит для пилотов. |
| 🧪 **По запросу** | В разработке | Тип заведён, распознавание есть, но качество не замерено. Включаем под конкретного клиента. |

## 3. Каталог поддерживаемых документов

«Таблицы/позиции» = умеет ли извлекать массив строк (товары/услуги/грузы) —
ключевой признак для счетов, накладных, спецификаций.

### 3.1. 🟢 Production — готовые типы

| Документ | Что извлекаем | Таблицы/позиции |
|---|---|---|
| **Счёт на оплату** | номер, дата, продавец/покупатель (ИНН/КПП/банк), суммы, НДС, позиции | ✅ да |
| **Счёт-фактура** | реквизиты сторон, НДС по ставкам, позиции | ✅ да (несколько ставок НДС) |
| **УПД** (универсальный передаточный) | реквизиты сторон, суммы, НДС, позиции | ✅ да |
| **Товарно-транспортная накладная (ТТН)** | отправитель/получатель/перевозчик, ТС, водитель, маршрут, груз | ✅ да |
| **CMR** (международная накладная) | отправитель/получатель/перевозчик, страны, груз, места погрузки/разгрузки, Incoterms | ✅ да |
| **Акт выполненных работ / услуг** | стороны, суммы, НДС, перечень услуг, период | ✅ да |

> По этим типам есть типизация и валидаторы; критичные поля (ИНН сторон, номер,
> сумма, госномер) дополнительно проверяются (контрольная сумма ИНН, формат
> госномера, согласованность НДС).

### 3.2. 🔵 Beta — расширенная поддержка (платёжные, ВЭД, договорные)

| Документ | Что извлекаем | Таблицы/позиции |
|---|---|---|
| **Платёжное поручение** | плательщик/получатель (ИНН/КПП/счёт/БИК), сумма, назначение | — (платёж, без таблицы) |
| **Кассовый чек (ККТ, 54-ФЗ)** | ФН/ФД/ФП, продавец, позиции, способ оплаты | ✅ да |
| **Commercial Invoice** (ВЭД) | exporter/consignee, валюта, Incoterms, позиции с HS-кодами | ✅ да |
| **Proforma Invoice** | стороны, валюта, условия, позиции | ✅ да |
| **Packing List** (упаковочный лист) | места, вес нетто/брутто, объём, позиции | ✅ да |
| **Коносамент (Bill of Lading)** | стороны, судно/рейс, порты, контейнеры, груз | ✅ да (контейнеры) |
| **Таможенная декларация (ДТ/ГТД)** | декларант/отправитель/получатель, графы, товары с ТН ВЭД | ✅ да |
| **Сертификат происхождения** (СТ-1/Form A/E) | стороны, страна происхождения, ТН ВЭД | — |
| **Сертификат/декларация соответствия ЕАЭС (EAC)** | номер, заявитель, изготовитель, ТР ТС/ЕАЭС | — |
| **Заявление на перевод в валюте (ВЭД)** | стороны, SWIFT/IBAN, валюта, назначение | — |
| **Акт взвешивания контейнера** | контейнер, веса (брутто/тара/нетто), порт/весы | — |
| **Прайс-лист** | поставщик, валюта, позиции (артикул/цена) | ✅ да |
| **Договор** | стороны (ИНН/роли), предмет, сумма, сроки | — (реквизиты, не текст) |
| **Доп. соглашение** | ссылка на договор, стороны, перечень изменений | — |
| **Спецификация / приложение к договору** | ссылка на договор, стороны, позиции, суммы | ✅ да |
| **Перемещение товаров (ТОРГ-13)** | организация, склады, позиции | ✅ да |

### 3.3. 🧪 По запросу — в разработке

| Документ | Что извлекаем | Таблицы/позиции |
|---|---|---|
| **Заявка на перевозку** | заказчик/перевозчик, маршрут, груз, ТС, ставка | частично |
| **Транспортная накладная (форма 2013)** | стороны, ТС, водитель, маршрут, груз, стоимость | частично |
| **Путевой лист** (4-С/4-П/ПЛ-1) | организация, ТС, водитель, маршрут, топливо, осмотры | — |
| **Корректировочный УПД (УКД)** | ссылка на исходный, стороны, суммы до/после | — |
| **Доверенность (М-2/М-2а)** | доверитель, представитель, срок, перечень ТМЦ | ✅ да |
| **Приём-передача ТМЦ на хранение (МХ-1)** | поклажедатель/хранитель, позиции | ✅ да |
| **Возврат ТМЦ с хранения (МХ-3)** | поклажедатель/хранитель, позиции | ✅ да |
| **Требование-накладная (М-11)** | отправитель/получатель, склад, позиции | ✅ да |

➕ **Пользовательские типы.** Под индивидуальную схему полей клиента заводим
свой тип документа — без изменения кода.

## 4. Возможности сервиса (для блока «Почему мы»)

| Возможность | Детали |
|---|---|
| **Форматы** | PDF, JPG, PNG, XLSX, DOCX (TIFF/HEIC — по согласованию) |
| **Размер / объём** | до 50 МБ, до 30 страниц |
| **Языки** | русский, английский (мультиязычные ВЭД-документы) |
| **Таблицы/позиции** | извлечение массива строк с количеством, ценой, НДС, весами, кодами ТН ВЭД |
| **Многодокументные файлы** | один PDF с несколькими документами разбивается автоматически |
| **Проверка качества** | контрольная сумма ИНН, согласованность НДС, форматы дат/госномеров/счетов |
| **Честность** | если поле не читается — возвращаем «пусто», а не выдуманное значение |
| **Достоверность** | оценка confidence по каждому полю; спорное уходит «на проверку оператору» |
| **Интеграция** | REST API + webhook (результат прилетает по готовности), подпись HMAC |
| **Безопасность** | передача по TLS, авторизация по токену, обезличивание ПДн по запросу |
| **On-prem / 152-ФЗ** | возможен полностью локальный контур на собственных моделях (без облака) |

## 5. Качество (формулировки для сайта — осторожные)

> ⚠️ Внутренняя пометка: точность замерена на ограниченном наборе и по
> production-типам; per-тип цифры по beta/experimental **не публиковать как
> гарантию**. Безопасные формулировки ниже.

- «Тип документа, номер, дата и таблица позиций распознаются стабильно».
- «Критичные реквизиты (ИНН сторон, суммы, госномер) дополнительно проверяются
  на корректность».
- «Точность растёт на ваших данных — пилот калибрует модель под ваш
  документооборот».
- Числовые SLA (скорость/точность) давать **как целевые, поэтапно** — не как
  жёсткую гарантию (см. договорные формулировки в `CONTRACT_TECH_APPENDIX.md`).
