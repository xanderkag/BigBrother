# Что нужно от parsdocs для сверки документов (SLAI)

Привет! По сверке документов в SLAI — вы уже извлекаете почти всё, что нужно, и
это всё завелось в сверку:

- у **ДТ** — построчные товары (гр.31: наименование, ТНВЭД, кол-во, вес нетто/брутто,
  стоимость, страна) + гр.44 (список документов, на которые ссылается декларация);
- у **спецификации** — шапка (№ контракта, Incoterms, валюта, стороны, суммы) + позиции;
- у **пакинга** — позиции с весами, объёмом, местами;
- **акт взвешивания** — взвешенный вес (его уже используем, ловим перевес).

Спасибо, реально многое работает. Уперлись, по сути, в одно место — **вес по
контейнерам**. Ниже конкретно.

---

## Главное: вес / объём / места ПО КОНТЕЙНЕРАМ

Сейчас `containers` приходит списком номеров:

```json
"containers": [ { "number": "CAIU7958712" }, { "number": "FESU5360805" } ]
```

Для заказов с несколькими контейнерами нам нужно знать вес/объём/места по
**каждому** контейнеру, а не только общий тотал по документу. Если в документе
(обычно это пакинг-лист или контейнерная разбивка в коносаменте) есть табличка
«по контейнерам» — извлекайте её, например так:

```json
"containers": [
  { "number": "CAIU7958712", "gross_weight_kg": 14000, "net_weight_kg": 12000, "volume_m3": 33.1, "packages": 679 },
  { "number": "FESU5360805", "gross_weight_kg":  8000, "net_weight_kg":  6800, "volume_m3": 25.9, "packages": 361 }
]
```

(поля опциональные — что реально есть в документе, то и кладите).

Без этого не собрать пер-контейнерную сверку и контрольную связку
«Σ по контейнерам = итог заказа = итог в ДТ». Весь остальной каркас у нас готов —
ждём только эти данные.

**Вопрос к вам:** из каких документов это реально достаётся? По вашему опыту, где
чаще всего есть разбивка веса/мест по контейнерам — пакинг, коносамент, что-то ещё?

---

## Заодно, помельче (не блокеры)

- **Позиция → контейнер.** Если в пакинге/BL видно, какой товар в каком контейнере,
  добавьте в позицию `container_number`. Тогда сможем разложить товары по
  контейнерам. Не срочно — многоконтейнерных заказов у нас пока мало.

- **Координаты значений (bbox).** Если сможете возвращать, где на странице лежит
  извлечённое значение, — сделаем подсветку прямо в просмотрщике документа
  («вот здесь этот вес»). Дальний бэклог, не блокер.

## Не про поля — про модель (если ещё актуально)

Раньше большие ГТД спотыкались на распознавании: «maximum context length»
(контекст модели 8192 токена, длинная декларация не влезала). Если крупные
декларации всё ещё не долетают — надо поднять `--max-model-len` до ~16k на vLLM
(10.10.33.10:8100). Дайте знать, если уже поправили — тогда снимаем вопрос.
