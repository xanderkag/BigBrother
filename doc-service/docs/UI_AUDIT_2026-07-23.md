## Дыры — приоритизировано

### P0 · реализуемо сейчас (бэкенд готов)

**1. Редактор ставок провайдера.** BILL-1 добавил `provider_settings.rates`, но менять их
можно только SQL/миграцией — в `Providers.tsx` редактора нет. Форма провайдера должна
показывать и править `rates`: `currency`, `cost_basis` (vendor/amortized/free), ставки
(`llm_input_per_1k`, `llm_output_per_1k`, `ocr_page`, `ocr_page_table`). CRUD провайдеров
в API уже есть → это чистый UI. Без него смена тарифа = ручной SQL на боксе (я так и делал
для yandex-ai-studio).

**2. `cost_breakdown` в деталке задачи.** Снимок себестоимости per-job (BILL-1) в БД есть,
но `JobDetail.tsx` его не раскрывает. Нужна строка/панель: построчно провайдер × вид расхода
× ставка × сумма, плюс метка `estimate`, плюс `cache_hit`/`would_cost` для кэш-хитов.

### P0 · нужен бэкенд (BILL-5), потом UI

**3. Бюджет по инстансу/оргe — то, что просил владелец.** Сейчас бюджет только на
gateway-consumer (чат, `Integrations`). После tenant split нужен **месячный бюджет на оргу**
(`slai-bctt`/`slai-app`/…) с порогами предупреждения — по макету 2026-07-23. Это BILL-5:
таблица `budgets` + `budget_events` + API. Как только бэкенд есть — форма установки бюджета
(лимит, период, пороги; «жёсткая блокировка» — заглушка, в этой версии только предупреждаем)
и список инстансов с прогресс-баром.

### P1 · сводка и видимость

**4. Экран «Расходы» (BILL-6).** Отдельного пункта меню `Расходы/Биллинг` нет. Нужен: сводка
по разрезам (орга/проект/тип/провайдер/день), график по дням, плашки бюджетов, строка
«сэкономлено кэшем», индикатор доли `estimate`. Нужен BILL-2 API (`GET /billing/summary`).
Под логином тенанта — тот же экран, но только его данные (ТЗ §5.3).

**5. Обзор тенантов в `Organizations`.** Провижн-модалка есть, но нет обзорной таблицы:
по каждой оргe — `webhook_url`, отпечаток токена, последняя активность, расход за период.
После сегодняшнего split это первое, на что смотрят («кто куда шлёт и сколько тратит»).

### P2

**6. Выдача gateway-ключей по инстансу (BILL-4).** Чтобы завести чат-ключ на стенд из UI и
увидеть `caller`-разрез расхода ассистента по инстансу (сейчас всё под общим `parsdocs`).
