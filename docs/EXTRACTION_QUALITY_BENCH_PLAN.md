# План: бенч качества извлечения (модели + охват)

**Дата:** 2026-06-18
**Повод:** замер на насыщенном УПД показал, что текущий прод-движок (`qwen2.5vl:7b`)
вытаскивает ~20% полей богатого документа (теряет реквизиты сторон сверх ИНН,
банк, ОГРН, подписантов и всю таблицу позиций). Нужен честный сравнительный
замер моделей по **охвату** и точности, чтобы выбрать движок в прод.

## Цель
Найти движок (+ подсказку/схему), который максимально полно и точно извлекает
из документов. Главная метрика — **охват** (% присутствующих полей, которые взяли).

## Метрики
1. Охват — % присутствующих полей, извлечённых.
2. Точность — % извлечённого, совпавшего с эталоном.
3. Критичные поля отдельно — ИНН/КПП, суммы/НДС, даты, номер.
4. Позиции — верное число строк (без дублей), полнота колонок.
5. Реквизиты-запас — банк / ОГРН / подписанты, когда есть.
6. Скорость — сек/документ.
7. Чистота — нет заглушек «не указано», нет выдуманных значений.

## Кандидаты (реально подняты на 33.10)
- Базовый (текущий прод): `qwen2.5vl:7b`.
- Чемпион текста: `mistral-small3.1` (24B).
- Тяжёлые: `qwen2.5:72b`, `llama3.3:70b`.
- Средние: `gemma3:27b`, `qwen3:32b`.
- РФ-специализированные: `T-pro-it` (Т-Банк), `RuadaptQwen2.5-32B` (МГУ).
- Vision (сканы): `qwen2.5vl:72b` vs `:7b`.

## Тестовый набор
- A. «Потолочные» доки — по 1 на тип (УПД/счёт/коммерческий инвойс/ТТН/акт)
  со ВСЕМИ полями, с известным эталоном → охват против «потолка».
- B. Обычные синтет-доки (текущий corpus) — базовая линия.
- C. Реальные с эталоном (golden-set) — честная точность.
- D. Разное качество: чистый текст-PDF vs скан.

## Как гоняем — два пути
1. Быстрый отбор: модель напрямую на 33.10 (минуя шлюз) + текст документа +
   ceiling-инструкция → JSON → охват. Без трогания прода, отсеиваем 2–3 финалистов.
2. Полный end-to-end: финалистов через реальный конвейер (переключение инференса,
   нужен Александр) — по одной модели.

## Что ещё варьируем
- Подсказка: текущая vs усиленная (запрет заглушек + список реквизитов/подписантов).
- Схема: текущая vs обогащённая (банк/ОГРН/подписанты).
- Один проход vs multipass (+ дедуп, уже готов).
- Текст vs картинка на сканах.

## Критерии выбора
Охват ≥ цель (~85%) на «потолочных» · критичные ≥ 95% · позиции 100% без дублей ·
в SLA по времени · 0 заглушек/выдумок · РФ-реквизиты ловятся.

## Этапы
1. Корпус «потолок» (5 насыщенных доков + эталон) + golden-set.  ← в работе
2. Метрика охвата в замерочный инструмент.
3. Быстрый A/B через прямой вызов 33.10: ~8 моделей × корпус → таблица.
4. Финалисты — полный end-to-end + сравнение с базовым 7b.
5. Vision-ветка (7b vs 72b на сканах).
6. С победителем — A/B подсказки и схемы.
7. Свод: таблица «модель × метрики» + рекомендация + «было 20% → стало X%».
