# ТЗ: Обработка разнообразных входящих файлов в parsedocs

> ⚠ **Не путать с интеграцией SLAI/ERP.** Этот документ — про **приём входящих
> файлов** (форматы, конвертация, приведение к каноническому виду), а не про
> интеграцию с Суперлогистом. Детальные требования по каждому формату —
> [`FILE_TYPES_SPEC.md`](FILE_TYPES_SPEC.md). *(Переименован из `INTEGRATION_TZ.md`
> 02.06 — старое имя ложно намекало на «интеграцию».)*

**Дата:** 15 мая 2026
**Статус:** черновик
**Главная проблема:** документы которые присылают сотрудники и контрагенты — **очень разные**. Один файл может быть нормальным PDF с векторным текстом, следующий — фото с iPhone в HEIC, третий — архив с 50 jpg по странице, четвёртый — email с вложениями. Система должна понять что пришло, привести к рабочему виду и обработать.

---

## Реальный зоопарк входящих файлов

### По формату

| Формат | Откуда берётся | Что с ним делать |
|--------|----------------|------------------|
| **Native PDF** (vector text) | 1С, ERP, Excel-экспорт, бухпрограммы | Извлечь текст напрямую (мгновенно) |
| **Scanned PDF** (текст как картинка) | МФУ, копир, скан-приставки | OCR постранично |
| **Mixed PDF** (часть текст, часть скан) | Подписанные документы (вёрстка + сканированная подпись) | OCR только страниц-картинок |
| **JPG / PNG / BMP** | Фото со смартфона, скриншот, отдельная страница | Один файл = одна страница |
| **TIFF многостраничный** | Корпоративные сканеры (Ricoh, Canon) | Раскладываем на страницы |
| **HEIC** | iPhone / iPad | Конвертация в JPG перед обработкой |
| **DOCX / XLSX** | Экспорт из Word/Excel, оригинал в редактируемом виде | Конвертация → PDF |
| **EML / MSG** | Forward письма с вложениями | Извлекаем `*.pdf`/`*.jpg` вложения |
| **ZIP / RAR / 7Z** | Архив документов за период | Распаковка → N jobs |
| **TIF-факс** | Старые системы факсимильной связи | Особый TIFF compression |

### По состоянию

| Проблема | Признак | Что делать |
|----------|---------|------------|
| **Зашифрованный PDF** | Запароленный, открывается с pin | Запросить пароль у клиента или отклонить |
| **Защищённый PDF** | Запрещён extract / print | Попробовать через OCR постранично (картинка обойдёт защиту) |
| **Повёрнутая страница** | Текст лежит на боку | Auto-rotate через OCR confidence |
| **Кривой скан** | Лист не выровнен, текст под углом | Deskew (поворот на 1-3°) |
| **Низкое разрешение** | <150 DPI, OCR confidence <50% | Предупредить оператора, deny на автоматическую обработку |
| **Цветной vs ч/б** | Цветной фон, печать жёлтым/красным | Binarize (преобразование в ч/б) если OCR не справляется |
| **Шум / артефакты** | Точки, полосы от старого МФУ | Denoise filter перед OCR |
| **Тонкая бумага** | Просвечивает оборот | Adaptive thresholding |
| **Битый файл** | Truncated PDF, corrupted JPEG | Понять и вернуть осмысленную ошибку, не падать молча |
| **Бесконечный размер** | Скан в 200 МБ, 30 страниц | Лимит + page-by-page обработка |

### По структуре содержимого

| Случай | Что внутри | Что делать |
|--------|------------|------------|
| **Один документ на N страниц** | ТТН на 2 листах (оригинал + копия), УПД на 3 листах (с приложением) | Обрабатывать как единое целое, итемы из всех страниц объединяются |
| **Склейка нескольких документов в один PDF** | Бухгалтер отсканировал пачку счетов в один PDF | Разделить на отдельные документы, создать N jobs |
| **Документ + сопроводительное** | Счёт + письмо менеджера на 1 странице, потом сам счёт на 2-й | Игнорировать письмо, найти бизнес-документ |
| **Дубликат страницы** | Скан случайно положил лист дважды | Detect + dedup |
| **Лишние пустые страницы** | Сепараторы из автоподатчика | Skip пустых |
| **Документ с приложениями** | ТТН + спецификация + сертификат соответствия | Решить: один job со всем или N jobs |

### По языку и содержимому

| Случай | Пример | Что делать |
|--------|--------|------------|
| **Русский** (типичный) | Российский счёт, УПД, ТТН | Tesseract `rus`, валидаторы РФ (ИНН/КПП) |
| **Английский** (ВЭД) | Commercial invoice, B/L | Tesseract `eng`, валидаторы ISO (страны, валюты) |
| **Смешанный RU+EN** | Российская компания с английскими графами в шапке | Tesseract `rus+eng` (так и настроено) |
| **Казахский / узбекский** | СНГ-партнёры | Доп. языковой пак Tesseract (`kaz`, `uzb`) |
| **Китайский** (ВЭД с КНР) | Инвойсы от китайских поставщиков | `chi_sim`, и LLM обязательна для извлечения |
| **Рукописный текст** | Подпись, дата от руки, заметка маркером | OCR пропускает, vision-LLM может прочитать |

## Что умеем сейчас / чего пока нет

Актуальный статус по каждому формату (детект, edge-cases, «работает / не принимаем»)
— в [`FILE_TYPES_SPEC.md`](FILE_TYPES_SPEC.md) («Формат-за-форматом» + «Сводная таблица
состояний»). Кратко:

- **Принимаем:** PDF (vector + scan), JPG/PNG/BMP/TIFF/WebP, multipage PDF, auto-rotate
  (Tesseract OSD), языки `rus+eng`, лимит 50 МБ (`MAX_UPLOAD_MB`); тип файла — по magic-bytes.
- **Не принимаем (а в потоке есть):** HEIC, DOCX/XLSX, EML/MSG, ZIP/RAR, encrypted/multi-doc
  PDF; нет deskew/denoise, blank/dup-page detection, китайского, рукописного, vision-LLM на сканах.

## Pre-processing пайплайн (что нужно построить)

Новый слой **до** OCR-цепочки, 4 стадии:

1. **Detection** — magic-bytes (формат), encrypted-check, page count, тип содержания (vector/scan/mixed).
2. **Extraction & Conversion** — привести к canonical-страницам: PDF→pdftoppm/pdf-parse,
   HEIC→JPG, DOCX/XLSX→PDF, EML/ZIP→рекурсия по вложениям, encrypted→ask/reject.
3. **Quality enhancement** (для image-страниц) — auto-rotate, deskew, denoise, binarize, drop blank, dedup.
4. **Splitting** (multi-doc PDF) — маркеры `«СЧЁТ № »`/`«УПД № »`, blank-separator, LLM-classifier → N jobs.

Канонический интерфейс модуля (`PreprocessInput`/`PreprocessResult`/коды ошибок) и порядок
реализации — в [`FILE_TYPES_SPEC.md`](FILE_TYPES_SPEC.md).

## Сценарии бухгалтерии / логистики — типичные и редкие

### Типичные (80% потока)

1. **Счёт PDF от поставщика** — native PDF, 1 страница, текст vector. Обработка 2-3 сек на правилах.
2. **УПД сканированный** — PDF от МФУ, 1-2 страницы, OCR-cleaning + LLM на табличной части.
3. **ТТН с подписью** — отсканированный лист с печатью, qr-кодом, печатью. OCR + LLM.
4. **Платёжное поручение из Клиент-Банка** — native PDF, простое извлечение.
5. **Акт работ** — DOCX подписанный + PDF-печатная копия. Сейчас принимаем только PDF.

### Усложнённые (~15%)

6. **Пачка счетов за месяц** — один отсканированный PDF на 30 страниц = 30 счетов разных поставщиков. **Сейчас обработаем как один документ, всё перемешается.** Нужен splitter.
7. **iPhone-фото счёта** — HEIC. **Сейчас не примем.** Нужна конвертация.
8. **Email от менеджера** — `.eml` с вложенным PDF и неактуальной перепиской. **Сейчас не примем.** Нужен mail-parser.
9. **Архив документов за квартал** — ZIP с подпапками. **Сейчас не примем.**
10. **Документ из Контур.Диадок** — XML + PDF в одном пакете. XML содержит структурированные данные (которые уже извлечены!). **Сейчас игнорируем XML, парсим PDF — теряем время.**

### Редкие, но болезненные (~5%)

11. **Защищённый PDF от банка** — заpasswordен. **Падает.** Нужно UX чтобы запросить пароль.
12. **Криво отсканированный документ** — повёрнут на 90°, скан под углом. **Без deskew качество <50%.**
13. **Документ с водяным знаком** «КОПИЯ» / «DRAFT» крупным шрифтом поверх — OCR ловит водянку как текст.
14. **Многоязычный документ** — китайский invoice от поставщика. **Сейчас не умеем `chi_sim`.**
15. **Битый PDF** — недокачался по сети, последние страницы pусто. **Падаем с непонятной ошибкой.**

## Что предлагаем строить — приоритеты

Детальная разбивка по спринтам с оценками — в [`FILE_TYPES_SPEC.md`](FILE_TYPES_SPEC.md)

- **P0 (критично, ~1.5-2 нед):** HEIC→JPG, encrypted-PDF detection+UX, multi-doc splitter
  (маркеры `«СЧЁТ N»`/`«УПД N»`), auto-rotate+deskew, blank-page detection, DOCX/XLSX→PDF,

  splitter через LLM, smart-cropping фото, HEIC HDR, многослойный TIFF (CCITT-4/Group 4).

## Архитектура pre-processing'а

Сейчас: `file → OCR chain → classify → parse → validate → resolve`. Станет:
`file → [preprocess: detect → convert → enhance → split] → OCR chain → …`.
Новая папка `src/pipeline/preprocess/` (`detect.ts`, `convert.ts`, `enhance.ts`, `split.ts`);
каждый модуль принимает `OcrInput` + signals и возвращает новый `OcrInput`, массив (split)
или `failed` с кодом (напр. `password_required`).

Бинарные зависимости (libheif, libreoffice/unoserver-sidecar ~600 МБ, unrar, 7z, qpdf,
imagemagick/vips) и развилка по LibreOffice (sidecar vs deny) — в
[`FILE_TYPES_SPEC.md`](FILE_TYPES_SPEC.md) (per-format + §Решения).

## Чего НЕ должно быть в P0

- Vision-LLM (на нашем железе невозможно — отдельная задача GPU)
- Auto-correction OCR-ошибок ИИ-моделью (это уже LLM, фаза 2)
- Handwriting recognition (vision-LLM only)
- Smart-cropping фото (iPhone)
- Document fraud detection (фальсификации, водяные знаки)
