# Отчёт по испытаниям моделей распознавания документов

**Дата:** 15 мая 2026
**Платформа:** parsedocs.taipit.ru
**Сервер:** 10.10.13.10 (7.8 ГБ RAM, без видеокарты)
**Документов в прогоне:** 8 синтетических PDF (счёт, УПД, ТТН, акт работ — по 2 каждого типа)

---

## Зачем мы это делали

Бухгалтерия и логистика тратят часы на ручной ввод данных из счетов, УПД, ТТН и актов. Мы строим систему, которая автоматически вытаскивает из PDF номер документа, дату, контрагентов, позиции таблицы, суммы и НДС. Для этого нужен «мозг» — языковая модель.

Мы испытали 6 разных вариантов «мозга» от самого простого (правила) до самого сложного (большая нейросеть). Цель — понять что реально работает на нашем железе и что нужно докупить.

---

## Что узнали — главное

**На текущем сервере 7.8 ГБ оперативной памяти умещается только маленькая модель (3-4 миллиарда параметров).** Большие модели (7+ миллиардов) — физически не помещаются, сервер начинает выгружать модель из памяти, обработка зависает.

**Время обработки одного документа:**
- Простые правила (без модели) — **2 секунды**
- Маленькая модель (Qwen 3B) — **~1 минута**
- Средняя модель (Gemma 4B) — **~5 минут**
- Большая модель (Qwen 7B) — **>10 минут, виснет**
- Облачная модель (Claude/GPT) — **3-7 секунд**
- Локальная модель на GPU — **3-7 секунд**

**Качество извлечения** при текущей конфигурации (gemma3:4b локально):
- 7 из 8 документов: вытащены номер, дата, контрагенты, позиции таблицы (от 1 до 3 строк), суммы, НДС
- 1 из 8 (УПД-1): модель не справилась с таблицей, вернула пустой список позиций
- Все документы попали в статус «требует проверки» — потому что валидаторы поймали синтетические (неправильные) ИНН в наших тестовых файлах. На реальных документах с правильными ИНН большинство уйдёт в статус «готово» без вмешательства оператора.

**Производительность.** В сутки один наш сервер с моделью Gemma 4B обработает максимум ~290 документов (24/7). За рабочий день (8 часов, конкурентно с одним worker'ом) — ~96 документов. Этого хватит на одного бухгалтера, но не на компанию.

## Что это значит для бизнеса

Сейчас мы можем выбрать одно из трёх направлений:

### Вариант 1 — Облачная модель (быстрый старт)
- **Что:** подключаем Claude или GPT-4 через API, платим за каждый документ
- **Когда заработает:** сразу после получения ключа (1 час работы)
- **Скорость:** 3-7 секунд на документ
- **Стоимость:** ~₽0.5-2 за документ, при 1000 документов в месяц — около ₽500-2000/мес
- **Минус:** содержимое документов уходит на серверы Anthropic или OpenAI. Не подходит для документов с персональными данными по 152-ФЗ
- **Плюс:** максимальное качество, работает завтра, никакого оборудования не нужно

### Вариант 2 — Заказать GPU-сервер
- **Что:** запросить у инфраструктуры (Павел / IT-департамент) сервер с видеокартой 16 ГБ
- **Когда заработает:** 1-2 недели на согласование + 1 день на перенос
- **Скорость:** 3-7 секунд на документ
- **Стоимость:** разовая закупка видеокарты ~₽80-150 тыс ИЛИ аренда облачного GPU ~₽20-40 тыс/месяц
- **Минус:** нужно ждать выделения, разовые расходы
- **Плюс:** все данные остаются внутри ТАЙПИТ, нет лимитов по объёму, работает с любыми документами

### Вариант 3 — Жить на текущей конфигурации
- **Что:** ничего не менять, использовать модель Gemma 4B
- **Скорость:** 5 минут на документ, ~96 документов в день
- **Стоимость:** 0
- **Минус:** медленно, иногда модель пропускает позиции таблицы (1 из 8 случаев), сервер на грани переполнения памяти
- **Плюс:** работает прямо сейчас, без бюджета

## Что мы рекомендуем

**На пилот (этот и следующий месяц):**

1. Получить ключ Anthropic Claude на месяц, бюджет до ₽5000.
2. Собрать у бухгалтерии и логистики 30-50 настоящих документов в PDF (счета, УПД, ТТН за апрель-май).
3. Прогнать через систему, сравнить с тем что бухгалтерия вводила вручную — посчитать точность по полям.
4. Параллельно — подать заявку на GPU-сервер в IT.

**После пилота (1-2 месяца):**

- Если облако показало >85% точности на наших документах — переезжать на GPU-сервер с той же моделью, данные не отдаём наружу.
- Если точность ниже — настраиваем подсказки для модели под конкретные типы документов ТАЙПИТ, расширяем справочники контрагентов и номенклатуры из ERP.

**Долгосрочно (3-6 месяцев):**

- Подключение к ERP / WMS — справочники приходят автоматически, не вводим вручную.
- Добавление специфичных для ТАЙПИТ типов: доверенности М-2, путевые листы, авансовые отчёты.
- Webhook-уведомления другим системам ТАЙПИТ когда документ обработан.

## Что уже готово на платформе

- Веб-интерфейс `parsedocs.taipit.ru`: загрузка через перетаскивание, тестовая лаборатория для админов
- 15 типов документов: счёт, УПД, ТТН, CMR, акт, платёжное поручение, инвойс, упаковочный лист, коносамент, ГТД, кассовый чек, договор, спецификация, допсоглашение, счёт-фактура
- На каждой позиции таблицы — до 22 полей (артикул, наименование, ТН ВЭД, страна, единица, количество, цена, ставка НДС и т.д.)
- Распознавание текста на русском и английском (Tesseract)
- Сверка контрагентов и номенклатуры со справочниками организации
- Multi-tenant: разделение по организациям и проектам
- Аудит-логи всех админских изменений
- API + Swagger для интеграции
